## 前提条件

実装を始める前に、以下を確認してください。

| 項目 | 確認内容 |
|------|----------|
| スプレッドシート | 本番の求職者管理スプレッドシートを開けること |
| シート名 | 「連絡先一覧」という名前のシートが存在すること |
| 列名 | 1行目（ヘッダー）に「ID」列があること |
| 氏名列 | 1行目に「氏名」または「求職者氏名」列があること |
| API 情報 | 本番アプリの URL（例: `https://hoiku-crm.vercel.app`）が分かっていること |
| SYNC_API_KEY | Vercel の環境変数 `SYNC_API_KEY` の値を確認できること |
| 権限 | スプレッドシートと Apps Script の編集権限があること |

## 実装前の準備（バックアップ）

**本番環境に直接影響するため、必ずバックアップを取ってから実施してください。**

1. **スプレッドシートのコピーを作成**
   - 本番スプレッドシートを開く
   - メニュー **ファイル** → **コピーを作成**
   - 名前を「求職者管理 バックアップ 2026-XX-XX」のように日付付きで保存
   - このコピーは編集せず、実装前の状態として保管

2. **既存の GAS コードを保存（すでに GAS を使っている場合）**
   - 既存の `Code.gs` の内容をテキストエディタにコピーして保存
   - 問題が発生した場合に元に戻せるようにする

## 手順 1: サーバー側（Next.js）のデプロイ

電話番号＋氏名による重複防止のロジックはサーバー側（`sync-candidates.ts`）にあります。  
**本番のアプリをデプロイして、最新版を反映させてください。**

### 1-1. ローカルでビルド確認

```bash
cd /Users/asato/Desktop/hoiku-crm
npm run build
```

- エラーが出なければビルド成功
- エラーが出た場合は、その解消を優先してください

### 1-2. 変更をコミットしてプッシュ

```bash
git status
git add GAS_SYNC.gs src/lib/sync-candidates.ts
git commit -m "feat: ID自動付与・電話番号重複防止を本番へ"
git push
```

- Vercel に連携している場合は、プッシュ後に自動デプロイが開始されます
- Vercel のダッシュボードでデプロイが完了するまで待つ（通常 1〜3 分）

### 1-3. デプロイ完了の確認

- 本番アプリの URL にアクセスして、正常に表示されることを確認
- デプロイが完了していないと、後続の GAS の同期は新しいロジックで動きません

## 手順 3: GAS コードを貼り付ける

### 3-1. コードファイルを開く

- 左側のファイル一覧で **`Code.gs`** をクリック
- 既に別の名前の `.gs` ファイルがある場合は、そのファイルを開く

### 3-2. 既存コードを削除

- エディタ内のコードをすべて選択（Ctrl+A / Cmd+A）
- 削除（Delete または Backspace）

### 3-3. 新しいコードを貼り付け

1. このリポジトリの **`GAS_SYNC.gs`** を開く
   - パス: `/Users/asato/Desktop/hoiku-crm/GAS_SYNC.gs`
2. ファイルの内容をすべて選択してコピー
3. Apps Script のエディタに貼り付け（Ctrl+V / Cmd+V）

### 3-4. 保存

- **Ctrl+S**（Windows）または **Cmd+S**（Mac）で保存
- または画面上部の 💾 保存アイコンをクリック
- 保存後、プロジェクト名の横に「保存済み」と表示されることを確認

### 3-5. 確認ポイント

- コードの先頭に `/** hoiku-crm 求職者同期（Google Apps Script） */` のようなコメントがあること
- コードの末尾に `function onEditAutoAssignId(e) { ... }` があること
- 約 750 行程度のコードであること

## 手順 4: スクリプトプロパティを設定

GAS から API への接続に必要な情報を設定します。

### 4-1. プロジェクト設定を開く

1. 左サイドバーの **プロジェクトの設定**（⚙️ 歯車アイコン）をクリック
2. **スクリプト プロパティ** のセクションまでスクロール
3. **スクリプト プロパティを追加** をクリック

### 4-2. API_URL を追加

1. **プロパティ** に `API_URL` と入力（大文字小文字を正確に）
2. **値** に `https://あなたの本番アプリ.vercel.app/api/sync/candidates` と入力
   - 例: `https://hoiku-crm.vercel.app/api/sync/candidates`
   - 末尾の `/api/sync/candidates` を忘れないこと
3. **スクリプト プロパティを保存** をクリック

### 4-3. SYNC_API_KEY を追加

1. 再度 **スクリプト プロパティを追加** をクリック
2. **プロパティ** に `SYNC_API_KEY` と入力
3. **値** に Vercel の環境変数 `SYNC_API_KEY` と同じ値を入力
   - Vercel ダッシュボード → プロジェクト → Settings → Environment Variables で確認
   - 値はコピー＆ペーストし、前後のスペースを入れないこと
4. **スクリプト プロパティを保存** をクリック

### 4-4. 設定の確認

1. `checkSettings` を関数選択で選び、**実行** をクリック
2. **表示** → **ログ** を開く
3. 以下のように表示されれば OK:
   ```
   === 設定チェック ===
   API_URL: https://あなたのアプリ.vercel.app/api/sync/candidates
   SYNC_API_KEY: 設定済み（XX文字）
   接続テスト ステータス: 200
   ```

## 手順 5: 初回セットアップ（トリガー登録）

氏名入力時に ID を自動付与するトリガーを登録します。

### 5-1. setupAutoIdTrigger を実行

1. 上部の関数選択プルダウンで **`setupAutoIdTrigger`** を選択
2. **実行** ボタン（▶）をクリック

### 5-2. 権限の承認（初回のみ）

初回実行時は、Google が権限の承認を求めます。

1. ダイアログで **「権限を確認」** をクリック
2. 表示される Google アカウントを選択
   - 本番スプレッドシートを編集できるアカウントを選ぶ
3. **「このアプリは確認されていません」** と出た場合:
   - **「詳細」** をクリック
   - **「（プロジェクト名）に移動（安全ではないページ）」** をクリック
   - **「許可」** をクリック

### 5-3. 実行結果の確認

1. **表示** → **ログ** を開く
2. 以下のメッセージが出ていれば成功:
   ```
   ✅ 自動 ID 付与トリガーをセットアップしました
   → 「連絡先一覧」シートで氏名を入力した行の ID 列が空の場合、自動で連番 ID が付与されます
   ```

### 5-4. トリガーの確認（任意）

1. 左サイドバーの **トリガー**（⏰ 時計アイコン）をクリック
2. **onEditAutoAssignId** というトリガーが表示されていれば OK
3. イベント: 「スプレッドシートから」→「編集時」

## 手順 6: 既存データの空 ID を一括付与（任意）

**氏名はあるが ID が空の行** がある場合に実行します。  
ID が空の行がなければ、この手順はスキップして構いません。

### 6-1. 実行前の確認

- スプレッドシートの「連絡先一覧」で、氏名あり・ID 空の行がないか確認
- ある場合は、この手順を実行

### 6-2. autoAssignMissingIds を実行

1. 関数選択で **`autoAssignMissingIds`** を選択
2. **実行** をクリック
3. **表示** → **ログ** で結果を確認

### 6-3. ログの確認例

```
ID 付与: 行42  氏名=山田 太郎  → ID=20206100
ID 付与: 行43  氏名=佐藤 花子  → ID=20206101
=== 完了 === 2 件の ID を付与しました（氏名が空の行はスキップ）
```

### 6-4. スプレッドシートでの確認

- スプレッドシートに戻り、付与された ID が意図通りか確認
- 誤りがあれば、手動で修正してください

## 手順 7: 重複 ID の確認（任意・推奨）

同期前に重複 ID がないか確認することを推奨します。

### 7-1. validateSheetDuplicateIds を実行

1. 関数選択で **`validateSheetDuplicateIds`** を選択
2. **実行** をクリック
3. **表示** → **ログ** で結果を確認

### 7-2. 結果の見方

**重複がない場合:**
```
✅ シート内に重複 ID はありません
```

**重複がある場合:**
```
⚠️ 重複 ID 20206004: 行49（山田 太郎）, 行120（佐藤 花子）
⚠️ 重複 ID が 1 件見つかりました。syncNewCandidates の前に修正してください
```

### 7-3. 重複 ID の修正方法

1. ログに表示された行番号と氏名をメモ
2. スプレッドシートで該当行を開く
3. どちらか一方の ID を、シート内の最大 ID + 1 などの新しい値に変更
4. 重複が解消されたら、再度 `validateSheetDuplicateIds` を実行して確認

## 手順 8: 同期の実行と動作確認

### 8-1. syncNewCandidates を実行

1. 関数選択で **`syncNewCandidates`** を選択
2. **実行** をクリック
3. 実行が完了するまで待つ（行数によって数十秒〜数分）

### 8-2. ログの確認

**表示** → **ログ** で以下を確認:

| 表示内容 | 意味 |
|----------|------|
| `✅ 送信データ内に重複 ID なし` | 送信データに重複はなし |

| `送信行数: XX` | 送信した行数 |
| `--- エラー ---` | エラーがあった場合の詳細 |
| `行N: シートID XXX は電話番号が既存の YYY と一致します。シートのIDを YYY に修正することを推奨します。` | 電話番号＋氏名による重複検知（サーバー側安全網） |

### 8-3. アプリ側での確認

- 本番の hoiku-crm アプリにアクセス
- 求職者一覧が期待通りに更新されているか確認
- 重複が増えていないか確認

### 8-4. 自動 ID 付与の動作確認（任意）

1. スプレッドシートの「連絡先一覧」に戻る
2. 新しい行を追加し、**氏名** 列にテスト用の名前（例: 「テスト 太郎」）を入力
3. **ID** 列は空のままにする
4. 別のセルをクリックして編集を確定
5. 数秒以内に **ID** 列に連番が自動で入ることを確認
6. テスト用の行は、確認後削除して構いません

## 運用時の補足

| 機能 | 実行タイミング | 説明 |
|------|----------------|------|
| **syncNewCandidates** | 通常の同期時（毎回） | 求職者データを DB に同期するメイン処理 |
| **validateSheetDuplicateIds** | 同期前の確認（任意） | 重複 ID が気になる場合に実行 |
| **autoAssignMissingIds** | 過去の空 ID を一括修正したいとき（任意） | 新規担当者引き継ぎ時など |
| **setupAutoIdTrigger** | 初回のみ | 一度設定すれば永続。トリガー削除時のみ再実行 |
| **checkSettings** | 接続エラー時 | API_URL と SYNC_API_KEY の確認 |
| **backfillRegisteredAt** | 登録日の補完が必要なとき（任意） | DB の registered_at が空の人を一括補完 |
| **syncAllContactUpdate** | 連絡先・年齢の一括更新が必要なとき（任意） | 電話・メール・年齢などをシートの値で更新 |

## トラブルシューティング

### エラー: 「シート「連絡先一覧」が見つかりません」

**原因:** 本番スプレッドシートに「連絡先一覧」という名前のシートがない

**対処:**
1. スプレッドシートのシート名を確認
2. シート名が「連絡先一覧」と完全一致していない場合（例: 全角スペース、余分な文字）、「連絡先一覧」に変更する

---

### エラー: 「ID 列または氏名列（氏名/求職者氏名）が見つかりません」

**原因:** 1行目（ヘッダー）に「ID」「氏名」「求職者氏名」のいずれかがない

**対処:**
1. 1行目のヘッダーを確認
2. 「ID」「氏名」または「求職者氏名」の列名が完全一致しているか確認

---

### トリガーが動かない（氏名入力時に ID が自動付与されない）

**原因:** トリガーが登録されていない、または権限が切れている

**対処:**
1. `setupAutoIdTrigger` を再度実行
2. 左サイドバーの **トリガー** で `onEditAutoAssignId` が存在するか確認
3. 削除されている場合は、再度 `setupAutoIdTrigger` を実行

---

### API エラー（401 認証が必要です / 403 など）

**原因:** API_URL または SYNC_API_KEY が誤っている

**対処:**
1. `checkSettings` を実行してログを確認
2. API_URL を確認: 末尾が `/api/sync/candidates` になっているか
3. SYNC_API_KEY を確認: Vercel の環境変数と完全一致しているか
4. 前後にスペースが入っていないか確認

---

### ログに「シートのIDを〇〇に修正することを推奨します」と出る

**意味:** 電話番号＋氏名が既存の求職者と一致するが、ID が異なっている状態

**対処:**
1. ログに表示された行番号と推奨 ID を確認
2. スプレッドシートで該当行の ID を、推奨された ID に修正
3. 同一人物の重複登録を防ぐための案内です

---

### ログの確認方法

1. Apps Script エディタで **表示** → **ログ** をクリック
2. または **Ctrl+Enter**（Mac: **Cmd+Enter**）でログを表示
3. 実行後、ログが表示されない場合は、もう一度 **実行** をクリックしてから確認

## ロールバック（元に戻す場合）

問題が発生した場合の手順:

1. **スプレッドシートのコピー**（実装前のバックアップ）から、必要なデータを復元
2. **GAS のトリガーを削除:** 左サイドバー → トリガー → 各トリガーの右側の ⋮ → トリガーを削除
3. **GAS コードを元に戻す:** 実装前に保存した既存のコードを貼り付けて保存
4. **サーバー側:** 必要に応じて、以前のコミットに戻してデプロイ
