## 📍 画面要素の命名規則

### カード系
- `[機能名]カード`: 情報をまとめたカード（例: 「残り営業日カード」「予算達成状況カード」）
- `[機能名]エリア`: 複数要素を含む領域（例: 「ヘッダーエリア」「フィルターバー」）

### テーブル系
- `[データ名]一覧テーブル`: データを一覧表示するテーブル（例: 「求職者一覧テーブル」「面接一覧テーブル」）

### ボタン系
- `[アクション名]ボタン`: アクションを実行するボタン（例: 「新規登録ボタン」「編集ボタン」）

### タブ系
- `[タブ名]タブ`: タブ切り替え内の各タブ（例: 「選考状況タブ」「成約情報タブ」）

## 🔍 よく使う指示パターン

### パターン1: カードの修正
```
「[ページ名]の『[カード名]』の[要素名]を[変更内容]してください」
例: 「ダッシュボードの『予算達成状況カード』のAヨミ表示を修正してください」
```

### パターン2: テーブルの修正
```
「[ページ名]の『[テーブル名]』の[カラム名/行]に[変更内容]してください」
例: 「求職者管理画面の『求職者一覧テーブル』の各行に編集ボタンを追加してください」
```

### パターン3: タブ内の修正
```
「[ページ名]の『[タブ名]タブ』内の『[コンポーネント名]』を[変更内容]してください」
例: 「求職者詳細の『成約情報タブ』内の『入金日』フィールドを必須にしてください」
```

### パターン4: フィルター・検索の修正
```
「[ページ名]の『[フィルター名]』に[変更内容]してください」
例: 「求職者管理画面の『検索バー』に年齢での検索機能を追加してください」
```

## 🎨 UI要素の色分け

- **バイオレット/インディゴ**: プライマリアクション、アクティブ状態
- **グリーン/エメラルド**: 成功、成約、完了
- **オレンジ/アンバー**: 警告、重要（優先度A）
- **ローズ/レッド**: エラー、最優先（優先度S）、Aヨミ
- **スレート/グレー**: 通常情報、無効状態

## ⚡ よくある修正箇所

| 修正内容 | 画面・場所 |
|---------|-----------|
| ステータス表示 | 各画面のBadgeコンポーネント |
| フィルター追加 | 各一覧画面のフィルターバー |
| テーブルカラム追加 | 各一覧テーブル |
| カードのスタイル変更 | 各サマリーカード |
| ボタンの追加 | ヘッダーエリア、カードヘッダー |
| ダイアログ追加 | 各画面の編集機能 |

## 💡 指示時の注意点

1. **ページ名とURLを併記**: 複数のページで似た要素がある場合に誤解を防げます
2. **階層を明確に**: 「カード」→「セクション」→「要素」の順で指定
3. **状態を指定**: タブが開かれている状態、フィルター適用後の状態など
4. **視覚的な説明**: 「右上のボタン」「赤色のバッジ」など補足情報を追加
